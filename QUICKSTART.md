# Быстрый старт ApeVault

## Шаг 1: Установка зависимостей

```bash
npm install
```

## Шаг 2: Настройка Firebase

1. Создайте проект в [Firebase Console](https://console.firebase.google.com/)
2. Включите Firestore Database
3. Скопируйте конфигурацию проекта
4. Создайте файл `.env` в корне проекта:

```env
VITE_FIREBASE_API_KEY=ваш-api-key
VITE_FIREBASE_AUTH_DOMAIN=ваш-проект.firebaseapp.com
VITE_FIREBASE_PROJECT_ID=ваш-проект-id
VITE_FIREBASE_STORAGE_BUCKET=ваш-проект.appspot.com
VITE_FIREBASE_MESSAGING_SENDER_ID=ваш-sender-id
VITE_FIREBASE_APP_ID=ваш-app-id
```

## Шаг 3: Запуск приложения

```bash
npm run dev
```

Приложение будет доступно по адресу: http://localhost:5173

## Основные функции

### Management (Управление)

- **Добавление слотов:** Нажмите "Добавить слот", выберите дату и время
- **Выходные:** Кнопка "Выходной" → выберите дату
- **Больничный:** Кнопка "Больничный" → выберите дату (только сегодня + 2 дня)
- **Отпуск:** Кнопка "Отпуск" → выберите даты
- **Просмотр:** Переключайтесь между таблицей и недельным видом

### Earnings (Заработок)

- **Добавить заработок:** Нажмите "Добавить заработок"
- **Выберите слот:** Выберите слот за день
- **Введите суммы:** Заработок и сумма в пул
- **Несколько участников:** Отметьте галочку для общего заработка

### Rating (Рейтинг)

- Автоматический расчет рейтинга
- Обновление каждые 7 дней
- Цветовая индикация прогресса

## Решение проблем

### Ошибка подключения к Firebase
- Проверьте правильность переменных в `.env`
- Убедитесь, что Firestore включен в Firebase Console

### Ошибка авторизации
- Проверьте правильность логина и пароля
- Убедитесь, что данные соответствуют `TEAM_MEMBERS` в коде

### Слоты не сохраняются
- Проверьте правила безопасности Firestore
- Убедитесь, что коллекция `workSlots` существует

## Поддержка

При возникновении проблем:
1. Проверьте консоль браузера (F12)
2. Проверьте логи Firebase Console
3. Убедитесь, что все зависимости установлены
