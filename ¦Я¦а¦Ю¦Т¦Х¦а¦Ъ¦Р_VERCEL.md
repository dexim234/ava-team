# Проверка настроек Vercel

## Критически важно проверить:

### 1. Переменные окружения Firebase

**Откройте проект на Vercel и проверьте:**

1. Перейдите на [vercel.com](https://vercel.com)
2. Откройте ваш проект `apevaultteam`
3. Перейдите в **Settings** → **Environment Variables**
4. **Убедитесь, что добавлены ВСЕ переменные:**

```
VITE_FIREBASE_API_KEY = [ваш ключ]
VITE_FIREBASE_AUTH_DOMAIN = [ваш домен]
VITE_FIREBASE_PROJECT_ID = [ваш project ID]
VITE_FIREBASE_STORAGE_BUCKET = [ваш bucket]
VITE_FIREBASE_MESSAGING_SENDER_ID = [ваш sender ID]
VITE_FIREBASE_APP_ID = [ваш app ID]
```

5. **Проверьте, что переменные добавлены для:**
   - ✅ Production
   - ✅ Preview (опционально)
   - ✅ Development (опционально)

6. **Если переменных нет - добавьте их:**
   - Нажмите "Add New"
   - Введите имя переменной (например, `VITE_FIREBASE_API_KEY`)
   - Введите значение из вашего `.env` файла
   - Выберите окружения (Production обязательно!)
   - Нажмите "Save"

7. **После добавления переменных:**
   - Перейдите в **Deployments**
   - Нажмите на три точки у последнего деплоя
   - Выберите **"Redeploy"**
   - Подтвердите

### 2. Проверка работы сайта

**Откройте сайт в приватном режиме:**

1. Откройте браузер в **режиме инкогнито** (Ctrl+Shift+N)
2. Перейдите на https://apevaultteam-t7h5.vercel.app
3. **Должна открыться страница авторизации** (Login)
4. Если видите ошибку - проверьте консоль (F12)

### 3. Проверка консоли браузера

**Если сайт не работает:**

1. Откройте сайт
2. Нажмите **F12** (DevTools)
3. Перейдите в **Console**
4. Проверьте ошибки:

**Если видите:**
- `Firebase: Error (auth/api-key-not-valid)` → Проверьте `VITE_FIREBASE_API_KEY`
- `Firebase: Error (auth/network-request-failed)` → Проверьте `VITE_FIREBASE_AUTH_DOMAIN`
- `Failed to fetch` → Переменные окружения не загружены, нужен Redeploy
- `Cannot read property 'env'` → Переменные не добавлены в Vercel

### 4. Проверка логов Vercel

1. Откройте проект на Vercel
2. Перейдите в **Deployments**
3. Откройте последний деплой
4. Проверьте **Build Logs** - должны быть без ошибок
5. Проверьте **Function Logs** - если есть ошибки, они будут здесь

## Быстрая диагностика:

### ✅ Всё работает, если:
- Сайт открывается
- Видна страница авторизации
- Можно ввести логин и пароль
- После входа открывается Management

### ❌ Проблема, если:
- Белая страница
- Ошибки в консоли браузера
- "Failed to fetch" или ошибки Firebase
- Переменные окружения не добавлены

## Решение проблем:

### Проблема: Белая страница

**Решение:**
1. Проверьте консоль браузера (F12)
2. Проверьте логи Vercel
3. Убедитесь, что переменные окружения добавлены
4. Сделайте Redeploy

### Проблема: Ошибки Firebase

**Решение:**
1. Проверьте все переменные окружения в Vercel
2. Убедитесь, что значения правильные (без кавычек, пробелов)
3. Сделайте Redeploy после добавления переменных

### Проблема: Другие пользователи не могут войти

**Это нормально!** Каждый пользователь должен:
1. Открыть ссылку
2. Ввести СВОЙ логин и пароль
3. Войти в систему

Данные авторизации хранятся в localStorage браузера, поэтому каждый должен авторизоваться самостоятельно.
